# insider-oracle

**動的専門家エージェント知見抽出システム**

ユーザーの質問に対し、その分野の最前線にいる複数の専門家視点を動的に編成し、一般には出回らないインサイダー級の知見を構造化して出力します。

## 特徴

- **動的エージェント編成**: 質問のドメインに応じて3〜5名の専門家を自動選定
- **インサイダー視点**: 「なぜその人が知っているか」の文脈を重視し、回答の深度を最大化
- **特化モジュール**: サラリーマン不動産投資節税に特化した4名固定エージェント搭載
- **知識蓄積**: 回答履歴・ドメイン別ナレッジを自動保存

## 使い方

Claude Code でこのリポジトリを開き、質問するだけで動作します。

```bash
# 汎用質問（エージェントが動的に編成される）
/ask 中小企業が銀行融資を通りやすくするための、銀行員が実際に見ているポイント

# 不動産投資節税（特化モジュールが起動）
/ask サラリーマンが不動産投資で節税する方法

# シミュレーション
/simulate 年収800万 1棟アパート

# 経費判定
/check 物件視察のための新幹線代

# リスク評価
/risk 築古木造の短期減価償却スキーム
```

## ディレクトリ構成

```
insider-oracle/
├── CLAUDE.md                  ← システムプロンプト（コマンドルーター）
├── README.md                  ← このファイル
├── agents/                    ← エージェント定義
│   ├── agent_a_tax_inspector.md
│   ├── agent_b_tax_accountant.md
│   ├── agent_c_bank_officer.md
│   └── agent_d_landlord.md
├── knowledge/                 ← 蓄積知識ベース
│   ├── tax/
│   ├── legal/
│   ├── realestate/
│   └── tech/
├── outputs/                   ← 生成されたMD回答
└── state/                     ← セッション状態
    ├── session.json
    └── agents_used.json
```

## 対応ドメイン

| ドメイン | エージェント編成 | 備考 |
|----------|------------------|------|
| 税務・財務 | 動的 + 特化モジュール | 不動産投資節税は4名固定 |
| 法律・規制 | 動的 | |
| 医療・健康 | 動的 | |
| 不動産・投資 | 動的 + 特化モジュール | |
| テクノロジー・AI | 動的 | |
| その他 | 完全動的 | 質問に応じて自動生成 |

## 設計思想

> エージェントの「肩書き」より「なぜその人が知っているか」の文脈が、回答品質を劇的に上げる。

このシステムの核心は、専門家の立場・経験・アクセスできた情報源を具体的に記述することで、一般的な回答ではなく、その立場だからこそ知りうる深い知見を引き出す点にあります。
